# CrawlerTennis 仓库功能与调用方式

本文档基于当前仓库代码整理，目标是说明这个仓库现在已经具备的能力、推荐入口、直接调用方式，以及哪些文件属于实验或遗留脚本。

## 1. 仓库定位

这个仓库当前的主用途是自动化操作复旦体育场馆预约系统，重点覆盖：

- 查询“江湾室外体育场-网球”的可预约时段
- 查看当前账号下的已预约记录
- 按序号、时间或关键字取消预约
- 按“距离开场还有多久”自动取消预约
- 在指定时间点并发触发预约脚本
- 直接执行江湾室外网球的预约动作

除上述主线外，仓库里还保留了一些实验性质的 Playwright 脚本，以及一个与网球预约无关的图片下载脚本 `craw_pinterest.py`。

## 2. 推荐入口

当前最统一、最适合直接使用的入口是根目录脚本：

```bash
./tennis_craw.sh
```

它把查询、查看预约、取消、定时取消、预约、定时并发执行等动作都封装成了子命令。

### 2.1 命令总览

```bash
./tennis_craw.sh query-jw [--json] [--headed] [--only-available]
./tennis_craw.sh list [--json] [--headed]
./tennis_craw.sh cancel <序号|关键字> [--yes] [--headed]
./tennis_craw.sh cancel --time "2026-04-03 18:00" [--yes] [--headed]
./tennis_craw.sh cancel-soon [提前量] [--headed]
./tennis_craw.sh cancel-loop [提前量] [检查间隔] [--headed]
./tennis_craw.sh book <day_index> <first|medium|back>
./tennis_craw.sh run-now [spec_path]
./tennis_craw.sh run-at <HH:MM[:SS]> [spec_path]
./tennis_craw.sh examples
```

### 2.2 子命令说明

#### `query-jw`

查询“江湾室外体育场-网球”的当前可预约时段。

示例：

```bash
./tennis_craw.sh query-jw
./tennis_craw.sh query-jw --only-available
./tennis_craw.sh query-jw --json
./tennis_craw.sh query-jw --headed
```

对应实现：

- Python 入口：`query_jiangwan_slots.py`
- Playwright spec：`tests/query_jiangwan_slots.spec.ts`

#### `list`

查看当前账号的已预约记录。

示例：

```bash
./tennis_craw.sh list
./tennis_craw.sh list --json
./tennis_craw.sh list --headed
```

对应实现：

- Python 入口：`list_reservations.py`
- Playwright spec：`tests/cancel_dry.spec.ts`
- 数据提取 helper：`tests/helpers/reservation.ts`

#### `cancel`

先读取当前预约列表，再按条件定位并取消目标预约。

支持三种定位方式：

- 按序号：`cancel 1`
- 按时间：`cancel --time "2026-04-03 18:00"`
- 按关键字：`cancel 江湾`

示例：

```bash
./tennis_craw.sh cancel 1
./tennis_craw.sh cancel 1 --yes
./tennis_craw.sh cancel --time "2026-04-03 18:00" --yes
./tennis_craw.sh cancel --keyword "5号场" --yes
```

说明：

- 默认会有确认提示
- 加 `--yes` 会跳过确认
- 匹配到多条记录时会直接退出，要求用户提供更精确条件

对应实现：

- Python 入口：`cancel_reservation.py`
- Playwright spec：`tests/cancel_by_time.spec.ts`
- 预约列表与取消逻辑：`reservation_client.py` + `tests/helpers/reservation.ts`

#### `cancel-soon`

取消“距离开始时间在指定提前量之内”的预约。

示例：

```bash
./tennis_craw.sh cancel-soon
./tennis_craw.sh cancel-soon 20m
./tennis_craw.sh cancel-soon 1h
```

说明：

- 默认提前量为 `60s`
- 支持 `s`、`m`、`h`
- 会先打印全部预约，再打印命中的即将开始条目，然后逐一取消

对应实现：

- Python 入口：`cancel_scheduler.py`

#### `cancel-loop`

周期性执行 `cancel-soon`，适合做简单轮询。

示例：

```bash
./tennis_craw.sh cancel-loop
./tennis_craw.sh cancel-loop 20m 5m
```

说明：

- 默认提前量 `60s`
- 默认检查间隔 `1h`
- 会循环调用 `cancel_scheduler.py`

对应实现：

- Python 入口：`cancel_scheduler_loop.py`

#### `book`

直接执行一次江湾室外网球预约。

示例：

```bash
./tennis_craw.sh book 0 first
./tennis_craw.sh book 1 medium
./tennis_craw.sh book 2 back
```

参数说明：

- `day_index`：候选日期索引，从 0 开始
- `slot_mode`：
  - `first`：选当天第一个可预约时段
  - `medium`：选中间位置的可预约时段
  - `back`：选最后一个可预约时段

对应实现：

- Python 入口：`book_tennis.py`
- Playwright spec：`tests/tennis_demo_jiangwan.spec.ts`
- 预约 helper：`tests/helpers/jiangwan_tennis.ts`

#### `run-now`

立即按并发配置启动一批 Playwright 预约任务。

示例：

```bash
./tennis_craw.sh run-now
./tennis_craw.sh run-now tests/tennis_demo_jiangwan.spec.ts
```

默认 spec：

```text
tests/tennis_demo_jiangwan.spec.ts
```

对应实现：

- Python 入口：`run_concurrent.py now <spec_path>`

#### `run-at`

在指定时间触发并发预约。

示例：

```bash
./tennis_craw.sh run-at 06:59
./tennis_craw.sh run-at 06:59:58
./tennis_craw.sh run-at 06:59 tests/tennis_demo_jiangwan.spec.ts
```

默认 spec：

```text
tests/tennis_demo_jiangwan.spec.ts
```

对应实现：

- Python 入口：`run_concurrent.py <HH:MM[:SS]> <spec_path>`

## 3. 直接调用方式

如果不走 `tennis_craw.sh`，也可以直接调用 Python 或 Playwright。

### 3.1 直接调用 Python 脚本

```bash
python query_jiangwan_slots.py --json
python list_reservations.py
python cancel_reservation.py 1 --yes
python cancel_reservation.py --time "2026-04-03 18:00" --yes
python cancel_scheduler.py 20m
python cancel_scheduler_loop.py 20m 5m
python book_tennis.py 2 back
python run_concurrent.py now tests/tennis_demo_jiangwan.spec.ts
python run_concurrent.py 06:59:58 tests/tennis_demo_jiangwan.spec.ts
```

### 3.2 直接调用 Playwright

#### 查询江湾空位

```bash
npx playwright test tests/query_jiangwan_slots.spec.ts --reporter=line
```

#### 查看已预约列表

```bash
npx playwright test tests/cancel_dry.spec.ts --reporter=line
```

输出里会打印一行以 `RES_DATA ` 开头的 JSON。

#### 取消预约

方式 1，按时间：

```bash
TARGET_TIME="2026-04-03 18:00" npx playwright test tests/cancel_by_time.spec.ts --reporter=line
```

方式 2，按序号：

```bash
TARGET_INDEX=1 npx playwright test tests/cancel_by_time.spec.ts --reporter=line
```

方式 3，传完整目标 JSON：

```bash
TARGET_RESERVATION_JSON='{"index":1,"time":"2026-04-03 18:00-19:00"}' \
npx playwright test tests/cancel_by_time.spec.ts --reporter=line
```

输出里会打印一行以 `CANCEL_RESULT ` 开头的 JSON。

#### 执行预约

```bash
DAY_INDEX=0 SLOT_MODE=first npx playwright test tests/tennis_demo_jiangwan.spec.ts
DAY_INDEX=1 SLOT_MODE=medium npx playwright test tests/tennis_demo_jiangwan.spec.ts
DAY_INDEX=2 SLOT_MODE=back npx playwright test tests/tennis_demo_jiangwan.spec.ts
```

## 4. 配置和环境变量

### 4.1 账号相关

Playwright helper 会优先读取以下环境变量：

- `BOOKING_USERNAME`
- `BOOKING_PASSWORD`

相关代码在：

- `tests/helpers/jiangwan_tennis.ts`
- `tests/helpers/reservation.ts`

注意：

- 当前代码里仍然保留了硬编码默认账号和密码作为兜底值
- 仓库虽然有 `.env` 文件，但 `playwright.config.ts` 中的 `dotenv` 加载逻辑是注释状态
- 也就是说，当前实现不会自动读取 `.env`，需要你自行 `export` 环境变量或在命令前内联传入

示例：

```bash
export BOOKING_USERNAME='你的账号'
export BOOKING_PASSWORD='你的密码'
./tennis_craw.sh list
```

### 4.2 预约选择相关

- `DAY_INDEX`：预约页的日期偏移索引，从 0 开始
- `SLOT_MODE`：`first | medium | back`

### 4.3 取消目标相关

- `TARGET_TIME`：按时间匹配，例如 `2026-04-03 18:00`
- `TARGET_INDEX`：按列表序号匹配
- `TARGET_RESERVATION_JSON`：传完整预约对象，优先级最高

### 4.4 并发配置

`run_concurrent.py` 会读取根目录的 `tennis_config.json`：

```json
{
  "day_config": [
    { "day_index": 2, "slot_mode": "back", "count": 3 }
  ]
}
```

字段含义：

- `day_index`：目标日期索引
- `slot_mode`：`first` / `medium` / `back`
- `count`：该组配置启动多少个并发线程

## 5. 当前核心文件分工

### 5.1 Python 主流程

- `tennis_craw.sh`：统一命令入口
- `query_jiangwan_slots.py`：查询江湾室外网球可预约时段
- `list_reservations.py`：读取当前已预约列表
- `cancel_reservation.py`：按条件取消预约
- `cancel_scheduler.py`：取消即将开始的预约
- `cancel_scheduler_loop.py`：循环执行取消检查
- `book_tennis.py`：执行一次预约
- `run_concurrent.py`：按配置并发触发 Playwright
- `reservation_client.py`：Python 侧对 Playwright 输出做封装，负责列预约和取消预约

### 5.2 Playwright 主流程

- `tests/helpers/jiangwan_tennis.ts`：登录、打开江湾预约页、查询空位、选择时段、确认预约
- `tests/helpers/reservation.ts`：打开“我的预约”、提取预约表格、匹配并执行取消
- `tests/query_jiangwan_slots.spec.ts`：输出江湾空位 JSON
- `tests/cancel_dry.spec.ts`：输出已预约列表 JSON
- `tests/cancel_by_time.spec.ts`：执行取消并输出结果 JSON
- `tests/tennis_demo_jiangwan.spec.ts`：实际预约用的主 spec

## 6. 实验、示例或遗留文件

以下文件目前不属于 `tennis_craw.sh` 的正式主链路，使用时要自己判断：

- `tests/test_thread.spec.ts`
  - 被 `run_concurrent.py` 的默认参数使用
  - 但当前代码里“确认预约”逻辑被注释掉了，实际更像并发选择时段测试，不是完整预约闭环
- `tests/tennis_demo.spec.ts`
  - 较早的手写流程，目标场馆和点击序号写死
- `tests/cancel.spec.ts`
  - 直接取消第一条预约的早期版本
- `tests/test-1.spec.ts`
  - 早期登录/跳转实验
- `tests/example.spec.ts`
  - Playwright 默认示例
- `e2e/example.spec.ts`
  - 不在当前 `playwright.config.ts` 的 `testDir=./tests` 范围内
- `craw_pinterest.py`
  - 与网球预约无关，是图片下载与去重聚类脚本

## 7. 运行前提

当前代码默认依赖以下环境：

- Python 3.10+
- Node.js
- 已安装 npm 依赖
- 已安装 Playwright Chromium 浏览器
- 运行机器可以访问 `https://booking.fudan.edu.cn`

补充说明：

- 代码中使用了 `str | None` 这类 Python 3.10+ 语法
- 当前仓库脚本默认调用的是 `python`，不是 `python3`
- 如果你的机器上 `python3` 仍然是 3.9，一些脚本和单元测试会直接报语法错误

常见准备命令：

```bash
npm install
npx playwright install chromium
```

如果要运行 Python 单元测试：

```bash
python -m unittest tests_python/test_reservation_client.py
```

## 8. 当前状态下的几个注意点

1. 仓库主能力已经形成“Python 封装 + Playwright 落地执行”的结构，但很多流程仍然直接依赖页面结构，页面变动会导致失效。
2. 统一入口推荐用 `tennis_craw.sh`，因为它对 `run_concurrent.py` 的默认 spec 做了更安全的覆盖，默认走的是 `tests/tennis_demo_jiangwan.spec.ts`。
3. 如果你直接运行 `python run_concurrent.py` 而不传 spec，它默认用 `tests/test_thread.spec.ts`，这个脚本当前不会点击“确认预约”。
4. 当前账号密码既可以走环境变量，也存在硬编码兜底值。从安全角度看，后续最好移除硬编码并启用显式配置加载。
